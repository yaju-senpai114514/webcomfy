ComfyUI 서버
체크 = 활성 (생성 대상 후보 + AFK 분산 기본값). ● 초록 = 응답 OK, M 배지 = 모델 관리 API 탑재. LoRA·체크포인트 파일은 모델 관리에서 서버별로 업로드/삭제/복사.
프롬프트 (Positive · 와일드카드)
각 블록: input 토큰(콤마 구분)이 프롬프트에 있으면 consume되고, 첫 등장 위치에 후보 한 줄이 매 생성마다 가중 랜덤 삽입됨. 표에서 — 체크박스 = 활성/비활성(주석), 가중치 열(0 이상 실수, 0이면 선택 안 됨), 후보 텍스트가 NOPROMPT면 consume만 하고 삽입 안 함. 블록별 복수 추출 (고급)을 펼치면 여러 후보를 가중 비복원 추출해 함께 삽입할 수 있으며, 기본 1개는 기존 동작과 동일. 블록 아래 + 하위 블록으로 이 블록이 치환된 뒤 이어서 굴려지는 하위 블록(재귀 트리)을 달 수 있음 — 예: __hair__ 블록의 후보 twintails, __len__가 뽑히면, 하위 __len__ 블록이 길이를 다시 굴림. 하위 블록도 다시 하위 블록을 가질 수 있음. 텍스트로 토글하면 후보 목록을 |n| 텍스트 원문으로 일괄 편집. 최상위 블록들은 좌우로 펼쳐져 나란히 표시됩니다.

AFK 정지됨
AFK는 매 생성마다 와일드카드/시드를 다시 굴려 정지할 때까지(또는 지정 개수까지) 백그라운드에서 저장한다. 브라우저를 닫아도 서버에서 계속 동작.
재현 (Reproduce)
저장된 webp에는 gz 압축된 config와 master seed가 RIFF 청크로 임베드된다. 위에 그 webp를 올리면 폼과 재현 시드를 복구한다. 같은 시드+config면 와일드카드·샘플러 시드까지 동일하게 재현된다.
